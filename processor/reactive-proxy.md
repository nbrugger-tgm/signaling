## Reactive Proxy Processor

Since Java in comparison to JS does not have dynamic objects highly dynamic typing (typescript),
to make life more convenient this processor helps you in creating complex reative objects.


### Gradle

To use this module you have to add an annotation processor dependency

```groovy
dependencies {
    annotationProcessor("eu.nitonfx.signaling:processor")
    implementation("eu.nitonfx.signaling:lib")
}
```

The code generated by the processor requires the inclusion of the lib itself

### Example / Showcase

Given you want to use a data object such as this in your UI model

```java
class TodoItem {
    boolean done;
    String text;
}
```
you cannot granularly update each field for fine-grained reactivity. By using `@Reactive` you can create a reactive complex signal
```java
@Reactive
interface TodoItem {
    boolean done();
    String text();
    void setText(String text);
    void setDone(boolean done);
}
```
which will create an implementation where each field is backed by a signal
```java
TodoItem item = ProxyFactory.create(cx, new TodoItem$Init("My task", false));
//or item = ProxyFactory.create(cx, TodoItem.class); with subsequent setting the fields
//or item = new ReactiveTodoItem(cx, "MyTask", false);
cx.createEffect(()->System.out.println("Task completion changed: "+item.done()));
cx.createEffect(()->System.out.println("Task text changed: "+item.text()));
item.setText("xyz"); //sout: Task text changed: xyz
item.setDone(true);//sout: Task completion changed: true
```

> I am actively searching for a solution for the first way of instantiation that does not include using an autogenerated class (`TodoItem$Init`) 

Should you happen to dislike code-generation (compile-time not runtime!) your best alternative is this

```java
record TodoItem(
        Signal<Boolean> done, 
        Signal<String> text
) {
}

TodoItem item = new TodoItem(
        cx.createSignal(false),
        cx.createSignal("MyTask")
);
cx.createEffect(()->System.out.println("Task completion changed: "+item.done.get()));
cx.createEffect(()->System.out.println("Task text changed: "+item.text.get()));
item.done.set(false);
item.text.set(true);
```
